## Lane Following vs Line Tracing

| 항목 | Line Tracing | Lane Following |
|------|-------------|----------------|
| 검출 대상 | 라인 1개 | 라인 2개 (좌+우) |
| 초기 위치 | 영상 중앙 | 좌: 1/4, 우: 3/4 |
| error 계산 | 영상중심 - 라인중심 | 영상중심 - (좌+우)/2 |
| ROI 분할 | 없음 | 좌우 분할 |

## 알고리즘 설명

### 자율주행 3대 기능
```
인지 → 판단 → 제어
```

### 전처리
1. **ROI 선정**: 하단 90px
2. **밝기 보정**: 평균 밝기 기준 자동 보정
3. **그레이스케일 변환**
4. **가우시안 블러**: 노이즈 제거
5. **이진화**: 평균+40 임계값으로 흰색 라인 분리

### 레인 검출 알고리즘

#### ROI 좌우 분할
```
전체 ROI (640px)
┌────────────┬────────────┐
│  좌측 ROI  │  우측 ROI  │
│  (0~320)   │ (320~640)  │
│ 초기: 1/4  │ 초기: 3/4  │
└────────────┴────────────┘
```

#### 라인 추적 (track_line)
- 각 ROI에서 컨투어 검출
- 노이즈 필터링 기준:
  - 면적 < 500 제거
  - width < 10 제거
  - height < 20 제거
  - height/width 비율 < 0.5 제거 (가로로 긴 노이즈 제거)
- 이전 위치에서 가장 가까운 후보 선택
- 정상 추적: 이동 제한 80px
- lost 상태: 제한 없이 재탐색

#### 라인 사라진 경우 처리
```
좌측 라인 사라짐 → 좌측 위치 = 우측 위치 - 레인폭
우측 라인 사라짐 → 우측 위치 = 좌측 위치 + 레인폭
양쪽 다 사라짐  → 이전 위치 유지
```

#### error 계산
```
mid_x = (좌측라인 x + 우측라인 x) / 2
error = 영상중심 x - mid_x
```

#### P제어 속도명령
```
좌측휠 = 직진속도 - k * error
우측휠 = -(직진속도 + k * error)
```

#### 설계변수
| 변수 | 기본값 | 설명 |
|------|--------|------|
| base_speed | 100 rpm | 직진속도 |
| k | 0.5 | P제어 게인 |
| min_speed | 50 rpm | 최소속도 |

#### 가감속 처리
- 모터 스레드 (50ms)
- ±5rpm씩 증감

## 터미널 출력
```
err:-10, lvel:95, rvel:-105, time:3.50msec
err:0,   lvel:100, rvel:-100, time:3.10msec
err:10,  lvel:105, rvel:-95, time:3.30msec
```

- `err`: 위치오차 (영상 중심 - 레인 중점)
- `lvel`: 좌측휠 현재 속도
- `rvel`: 우측휠 현재 속도
- `time`: 영상 1장 처리 시간 (30msec 이하 목표)

## 설명 문제

### Q1. Lane Following과 Line Tracing의 차이점을 설명하시오.

Line Tracing은 하나의 라인을 검출하여 그 라인을 따라가는 방식이다.
Lane Following은 좌우 두 개의 라인을 동시에 검출하여 두 라인 사이의
중점을 기준으로 레인 안에서 주행하는 방식이다.
Lane Following이 더 안정적이며 실제 자율주행에 가까운 방식이다.

### Q2. error 계산 방법을 설명하시오.
```
mid_x = (좌측라인 무게중심 x + 우측라인 무게중심 x) / 2
error = 영상 중심 x좌표 - mid_x
```
error > 0: 레인 중심이 왼쪽 → 우회전 필요
error < 0: 레인 중심이 오른쪽 → 좌회전 필요
error = 0: 레인 중심에 위치 → 직진

### Q3. 한쪽 라인이 사라졌을 때 처리 방법을 설명하시오.

한쪽 라인이 사라졌을 때 반대쪽 라인 위치와 레인 폭을 이용하여
사라진 라인의 위치를 추정한다.
좌측 라인이 사라지면 우측 라인 위치에서 레인 폭을 빼서 좌측 위치를 추정하고,
우측 라인이 사라지면 좌측 라인 위치에서 레인 폭을 더해서 우측 위치를 추정한다.

### Q4. P제어 게인(k)의 역할을 설명하시오.

k는 error에 대한 반응속도를 조절하는 게인이다.
k가 크면 같은 error에 대해 더 빠르게 방향을 전환하지만 진동이 발생할 수 있다.
k가 작으면 반응이 느려서 커브 구간에서 레인을 벗어날 수 있다.
직진속도가 클수록 k도 함께 증가시켜야 한다.

### Q5. 이진화 임계값 설정의 중요성을 설명하시오.

임계값이 너무 높으면 라인이 이진 영상에서 사라져 검출이 불가능하다.
임계값이 너무 낮으면 노이즈가 흰색으로 처리되어 잘못된 라인을 검출한다.
본 코드에서는 ROI의 평균 밝기에 40을 더한 값을 임계값으로 사용하여
조명 변화에 적응적으로 대응한다.
